## 実装ルール（全ページ共通）

### スタイル

- **`wireframe_top.html` の `<style>` セクションをそのままコピーして使う**（共通化のため）
- 色・フォント・画像はすべて仮。レイアウト確認のみが目的。
- ワイヤーは「箱と文字だけ」の思想を守る。装飾はしない。

### 共通要素（全ページに入れる）

すべてのページは以下の構造を持つ：

```
[ ページ情報バー（黒帯・page-meta）]
[ 赤帯（top-bar）]
[ ヘッダー（header）：ロゴ＋ナビ6項目 ]
[ ページタイトルエリア（ヒーローの簡易版）]
[ メインコンテンツ ]
[ CTAバナー（cta-band）]
[ フッター（footer）]
```

**ヘッダーナビ・フッターは全ページ共通の内容にする。**

### 注釈のルール

各ブロックに **オレンジ色の注釈**（`.wf-annotation` または同様のインライン注釈）を必ず入れる。
注釈の目的は「水戸さんがコーディング時に判断に迷わないようにする」こと。以下を明記：

- **実装要件**：「WPカスタム投稿」「Contact Form 7で実装」「iframeで埋め込み」など
- **素材の状態**：「クライアント素材待ち」「仮テキスト」「確定済み」
- **レスポンシブ挙動**：「スマホではハンバーガーメニュー」「スマホ1カラム」など

### ページ情報バーの内容

各ページの `.page-meta` には以下を記載：

```html
<div style="font-size:11px; opacity:0.7;">BNI岐阜 真チャプター サイトリニューアル</div>
<strong>ページ②：BNIとは・参加するメリット</strong>
<!-- 番号は各ページ固有 -->
```

## ページ別 実装指示

### ① wireframe_about.html（BNIとは・参加するメリット）

**ページ番号**：ページ②

**セクション構成**：

```
1. ページタイトル：「BNIとは ― Local Business, Global Network®」
2. BNIの概要（公式文言使用）
   - 世界最大級のビジネス・リファーラル組織の説明
   - チャプターの仕組み
   - Givers Gain®の理念
3. BNI紹介動画（YouTube埋め込み枠）
4. 参加するメリット（詳細版・6カード）
   - ビジネス紹介の増加
   - 経営者ネットワーク
   - スキル・プレゼン力向上
   - 定期的な学びの機会
   - 信頼関係の構築
   - ビジネスパートナーとの出会い
5. 数字で見るBNI（世界のメンバー数・年間リファーラル件数など ※仮数値）
6. CTAバナー
```

**注釈ポイント**：
- 概要テキストは公式文言を使用（元サイト・参考サイトと共通）
- メリット6項目のアイコンは Font Awesome または画像差し替え
- 数字部分は「クライアント確認要」と明記

### ② wireframe_faq.html（よくある質問）

**ページ番号**：ページ⑤

**セクション構成**：

```
1. ページタイトル：「よくある質問」
2. カテゴリ別FAQ（アコーディオン表示想定）
   - 入会について
   - 例会について
   - 費用について
   - その他
3. 「解決しなかったら」CTA → お問い合わせページへ
```

**FAQ項目の叩き台**（各カテゴリに3〜5問ずつ）：

#### 入会について
- Q. 誰でも入会できますか？
- Q. 入会までの流れを教えてください
- Q. 1業種1名という制限は本当ですか？
- Q. 見学はできますか？

#### 例会について
- Q. 毎週必ず参加しないといけませんか？
- Q. 例会ではどんなことをしますか？
- Q. オンライン参加はできますか？
- Q. 遅刻・早退はOKですか？

#### 費用について
- Q. 入会金・年会費はいくらですか？
- Q. その他にかかる費用はありますか？
- Q. 見学は無料ですか？

#### その他
- Q. リファーラルとは何ですか？
- Q. Givers Gain®とはどういう意味ですか？
- Q. BNIと他の異業種交流会の違いは？

**注釈ポイント**：
- 回答テキストは仮。クライアント確認後に確定。
- アコーディオンUI（クリックで開閉）を想定。JS実装が必要。
- 実装案：CSS `<details>` `<summary>` タグで最小実装可能。
